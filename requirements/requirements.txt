Mika Log — Full Requirements (Web + iOS)
Project type: Expo React Native (SDK 54)
OS target: macOS

==================================================
1) System prerequisites (install first)
==================================================
- Homebrew (latest)
- Node.js >= 20 (LTS recommended)
- npm >= 10
- Watchman (for reliable Metro file watching)
- Xcode (latest stable)
- Xcode Command Line Tools
- iOS Simulator runtime (installed via Xcode)
- CocoaPods >= 1.15 (needed if using prebuild/dev build)

Suggested install commands (macOS):
- /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
- brew install node watchman cocoapods
- xcode-select --install

Notes:
- Use `npx expo ...` (no global Expo CLI required).
- If you will run native builds (`expo run:ios`), keep Xcode + CocoaPods updated.

==================================================
2) Environment variables
==================================================
Create a `.env.local` file in project root with:

EXPO_PUBLIC_TMDB_API_KEY=your_tmdb_api_key_here

(Used by API config for movie/TV requests.)

==================================================
4) Project setup commands
==================================================
From project root:
- npm ci

Optional validation:
- npm run lint

==================================================
5) Run targets
==================================================
Web:
- npm run web
  (or: npx expo start --web)

iOS (Simulator):
- npm run ios
  (or: npx expo start --ios)

If using native dev build flow instead of Expo Go:
- npx expo prebuild --platform ios
- npx pod-install
- npx expo run:ios

==================================================
6) Troubleshooting baseline
==================================================
If Metro cache issues appear:
- npx expo start -c

If iOS native dependency sync is needed:
- npx pod-install

If Node modules are corrupted:
- rm -rf node_modules package-lock.json
- npm install
